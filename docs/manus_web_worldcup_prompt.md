2) 승자 결정 규칙(가중→raw→rank)과 4지선다→2매치 변환식을 임의 변경 금지. 웹 1:1 직접투표라도 IG표와 합산 시 동일 규칙으로 환산.
3) `post_ledger.json`의 `topic_id` 네이밍·조인키 역할 유지(확장 가능, 파괴 금지). 웹 투표 세션·집계 스냅샷·정정 이력은 같은 감사 추적에 새 `topic_id` 규칙으로 편입.
4) 멱등성(`already_done`)·동시성(`concurrency`)·캠페인 윈도우 가드 철학 유지. 웹이 라운드를 이중 전이시키지 않도록 설계.
5) 브랜드 안전 정책(클릭베이트 금지어·이모지 금지·미성년자 보호·신체비교 금지·정치정쟁 금지·저작권 변형·출처 표기)을 웹 카피·공유 OG·해시태그·유저생성 콘텐츠 전반에 그대로 적용.

■ 5. 출력 형식
────────────────────────
- 전부 한국어. 산출물 A → B → C 순서로 한 응답에 담되 각각 명확한 H1 제목으로 구분.
- 위 사실(파일명·스키마·`topic_id` 규칙·시드 방식·가중 집계·케이던스 등)을 구체적으로 인용해 일반론을 피할 것.
- 다이어그램은 머메이드 코드블록 우선. 표·체크리스트 적극 사용.
- 의사결정 갈리는 지점마다 옵션 2~3개 + 추천안·근거. 가정은 `[가정]`, 미결정은 `[결정필요]` 태그.
- 각 산출물 끝에 "기존 자동화와의 정합성 체크리스트"를 붙이고, 맨 마지막에 "오너에게 묻는 핵심 질문 5개 이내"를 덧붙일 것.
- 길되 군더더기 없이. 실행 가능성·구체성 우선.
